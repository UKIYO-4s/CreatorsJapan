## 提案概要: WordPress → Astro 移設

### なぜ Astro か？

| 比較項目 | WordPress（現状） | Astro + Cloudflare（提案） |
|----------|-------------------|---------------------------|
| **表示速度** | 遅い（サーバーサイド処理 + DB問い合わせ） | 超高速（静的HTML + Edge配信） |
| **セキュリティ** | 脆弱（プラグイン依存、DB攻撃面あり） | 堅牢（静的ファイル = 攻撃面ほぼゼロ） |
| **メンテナンス** | 高負荷（WP本体・プラグイン・PHP更新が常時必要） | 低負荷（更新不要、ビルド済みHTML） |
| **ランニングコスト** | 月額数千円〜（サーバー + ドメイン） | 基本無料（Cloudflare Free Tier） |
| **技術者不要度** | 低（WP専門知識が必要） | 高（Claude CLI で日本語指示だけで編集可） |
| **Lighthouse スコア** | 50〜70点（典型的なWPサイト） | 95〜100点 |
| **SEO効果** | 標準 | Core Web Vitals 大幅改善 → 検索順位向上 |

### アーキテクチャ

```
Browser
  ↓
Cloudflare Edge（世界300+拠点）
  ├ Astro Pages（静的HTML / SSR ハイブリッド）
  ├ Pages Functions / Workers（認証・API処理）
  ├ KV（セッション・キャッシュ）
  ├ D1（ユーザーDB・コンテンツメタデータ）
  ├ R2（動画・ファイルストレージ）
  └ Astro Content Collections（Markdown = WPの代替、外部CMS不要）
```

#### コンテンツ管理: なぜ外部CMSを使わないのか

| 方式 | 月額コスト | 運用方法 |
|------|----------|---------|
| microCMS 等の外部CMS | ¥4,900〜/月 | ブラウザで記事編集 → API配信 |
| **Astro Content Collections（本提案）** | **¥0** | **Claude CLI に指示 → Markdownファイル自動生成 → 自動公開** |

Astro にはMarkdownファイルをコンテンツとして扱う **Content Collections** 機能が標準搭載されています。
記事・講座・お知らせなどをすべて `.md` ファイルとしてGitリポジトリ内で管理します。

**Claude CLI との連携で外部CMSが不要に:**

```
┌─────────────────────────────────────────────────────────────┐
│  記事の追加・編集フロー                                       │
│                                                             │
│  担当者: 「動画編集の始め方という記事を追加して。              │
│           カテゴリは入門、タグは Premiere Pro と初心者」       │
│         ↓                                                   │
│  Claude CLI: src/content/blog/getting-started.md を作成      │
│         ↓                                                   │
│  自動: Git push → Cloudflare ビルド → 数分で本番公開         │
│                                                             │
│  ※ 外部CMS不要、月額費用ゼロ                                │
└─────────────────────────────────────────────────────────────┘
```

**メリット:**
- ランニングコスト ¥0（外部CMS契約が不要）
- コンテンツもGitで完全にバージョン管理（変更履歴が残る）
- 外部サービス障害の影響を受けない
- Claude CLI がある限り、記事管理に専門知識は不要

#### CLAUDE.md の設計方針（こちらで事前設定）

```markdown
# プロジェクトルール（CLAUDE.md に記載する内容の例）

## 変更可能な範囲
- src/pages/ 配下のページファイル（.astro）
- src/components/ 配下のUIコンポーネント
- src/styles/ 配下のスタイルファイル
- public/ 配下の画像・静的ファイル

## 変更禁止（触らないこと）
- src/lib/auth.ts（認証ロジック）
- src/middleware.ts（セキュリティミドルウェア）
- wrangler.toml（Cloudflare設定）
- functions/（APIバックエンド）

## コード変更時のルール
- 既存のデザインシステム（Tailwind CSS クラス）を使うこと
- 新しいnpmパッケージを追加しないこと
- 変更後は必ず git で feature ブランチにコミットすること
- コミットメッセージは日本語で変更内容を簡潔に書くこと
```
