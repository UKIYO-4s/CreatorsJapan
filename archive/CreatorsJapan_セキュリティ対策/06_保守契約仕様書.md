# 保守契約仕様書

## 1. SLA（サービスレベル）

### 営業時間

| 区分 | 時間 |
|------|------|
| 通常営業 | 平日 10:00 - 18:00 |
| 時間外 | 上記以外（土日祝含む） |

※ 年末年始（12/29-1/3）、GW、お盆は休業

---

### 緊急度定義と初動目標

| 緊急度 | 状況例 | 初動目標 |
|--------|--------|----------|
| **Critical** | サイトダウン、改ざん、情報漏洩の疑い | 営業時間内：2時間以内 / 時間外：翌営業日午前中 |
| **High** | スパム大量発生（10件以上/日）、主要機能停止 | 24時間以内 |
| **Medium** | 表示崩れ、軽微なエラー、更新依頼 | 3営業日以内 |
| **Low** | 質問、相談、改善提案 | 5営業日以内 |

**初動とは**: 状況確認を開始し、対応方針を連絡すること（解決完了ではない）

---

### 定期作業の納期

| 作業 | 頻度 | 納期 |
|------|------|------|
| スパム確認・処理 | 毎日 | 当日中 |
| プラグイン更新確認 | 週1回 | 毎週金曜まで |
| アクセスログ確認 | 週1回 | 毎週金曜まで |
| サイトヘルスチェック | 月1回 | 月末まで |
| 月次レポート提出 | 月1回 | 翌月5営業日まで |

---

## 2. 対応範囲

### 標準プランに含まれる作業

| カテゴリ | 内容 |
|----------|------|
| 監視 | スパム監視、サイトヘルス確認、アクセスログ確認 |
| 更新 | プラグイン更新（マイナー）、テーマ更新（マイナー） |
| 報告 | 月次セキュリティレポート |
| 軽作業 | 設定変更、テキスト修正（月2時間まで） |
| 相談 | セキュリティに関する質問対応 |

### 標準プランに含まれない作業（別途見積もり）

| カテゴリ | 内容 |
|----------|------|
| 開発 | 新機能追加、カスタマイズ |
| デザイン | レイアウト変更、画像作成 |
| 大規模更新 | WordPress本体メジャー更新、PHP更新 |
| インフラ | サーバー移行、ドメイン変更 |
| インシデント | 改ざん復旧、マルウェア除去（初動調査は含む） |
| コンテンツ | 記事作成、ページ追加 |

---

## 3. 監視指標

### スパム監視

| 指標 | 正常 | 注意 | 異常（通知） |
|------|------|------|--------------|
| 日次スパム件数 | 0-2件 | 3-9件 | 10件以上 |
| 新規パターン | なし | - | 未知のパターン検出 |
| 承認済み混入 | なし | - | 1件でも発生 |

**通知方法**: 異常検出時はメールで即時報告

---

### サイトヘルス

| 項目 | 正常 | 異常（通知） |
|------|------|--------------|
| サイトヘルススコア | 良好 | 致命的な問題あり |
| 更新待ちプラグイン | 0-3件 | 4件以上（セキュリティ更新は即時） |
| 停止中プラグイン | 0件 | 1件以上 |
| PHP/WPバージョン | サポート内 | サポート終了 |

---

### アクセスログ

| 確認項目 | 異常とみなす基準 |
|----------|------------------|
| 不正ログイン試行 | 同一IPから10回以上/日 |
| 404エラー急増 | 通常の3倍以上 |
| wp-login.php攻撃 | 100回以上/日 |
| xmlrpc.php攻撃 | 50回以上/日 |
| 不審なPOSTリクエスト | wp-comments-post.phpへの異常アクセス |

## 4. バックアップポリシー

### 保存期間

| バックアップ種別 | 頻度 | 保存期間 |
|------------------|------|----------|
| データベース | 毎日 | 30日分 |
| ファイル（全体） | 週1回 | 4世代（約1ヶ月） |
| ファイル（差分） | 毎日 | 7日分 |

※ サーバー側の自動バックアップと併用を推奨

---

### バックアップ確認

| 項目 | 頻度 |
|------|------|
| バックアップ実行確認 | 週1回（ログ確認） |
| 復元テスト | 半年に1回 |

---

### 復元手順（概要）

1. **状況確認** - 何を・いつ時点に戻すか決定
2. **作業告知** - クライアントへダウンタイム連絡
3. **事前バックアップ** - 復元前の現状を保存
4. **復元実行** - テスト環境で確認後、本番適用
5. **動作確認** - 主要機能のチェックリスト実行
6. **報告** - 復元完了レポート提出

**目標復旧時間（RTO）**: 4時間以内（営業時間内の場合）

## 5. エスカレーション

### 連絡経路

```
通常連絡 → メール / チャット
     ↓ 4時間応答なし
緊急連絡 → 電話
     ↓ 対応不可
代替対応 → [要設定：代替連絡先]
```

### クライアント側エスカレーション

| 状況 | 対応 |
|------|------|
| 初動目標超過 | 状況報告メールを送付 |
| 解決長期化（3日超） | 中間報告 + 見通し共有 |
| 対応困難 | 外部専門家への相談提案 |

---

## 6. 契約条件

### 最低契約期間

- 初回：6ヶ月
- 更新：1ヶ月単位（自動更新）

### 解約

- 1ヶ月前までに書面通知
- 解約月末で終了

### 免責事項

以下は保守対象外：
- 第三者サービスの障害（サーバー会社、CDN等）
- クライアント操作に起因する障害
- 天災、不可抗力による障害
- 契約外作業の結果生じた問題
