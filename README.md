# CreatorsJapan セキュリティ対策プロジェクト

## 概要

Creators Japan の WordPress サイト2件に対するセキュリティ診断・対策・継続監視プロジェクト。

**開始日**: 2026年1月11日

---

## 背景

会員制サイト（salon.creators-jp.com）にスパムコメントが発生。
2025年12月8日以降、カジノ・ギャンブル系スパムが7件投稿された。
会員制サイトに外部からスパムが侵入していることから、**セキュリティホールの存在が疑われる**。

| サイト | URL | 種別 | 状態 |

## プロジェクト目標

1. 既存スパムの処理
2. スパム発生原因（侵入経路）の特定

## 技術方針

### プラグイン使用方針

| 区分 | 方針 | 例 |
|------|------|-----|
| コア対策 | **プラグイン不使用** | REST API制限、XMLRPC無効化、セキュリティヘッダー |
| 認証強化 | プラグイン許容 | 2FA、ログイン試行制限 |
| 監視・検知 | プラグイン許容 | ログイン履歴、改ざん検知 |
| スパム対策 | 設定優先 | ブロックリスト、ディスカッション設定 |

### 実装方法

- `functions.php` によるREST API制限
- `.htaccess` によるXMLRPC無効化、セキュリティヘッダー
- サーバーパネルによるWAF設定

## 注意事項

### 海外会員への配慮

**海外在住の会員がいるため、一括での海外IPブロックは不可**

代替案：
- 管理画面（/wp-admin/）のみ日本IPに制限
- 特定の悪質な国/地域のみブロック
- BOT対策はWAF/Cloudflareで実施

### 作業時のリスク

- プラグイン/テーマ更新で表示崩れの可能性あり
- 作業前の**バックアップ必須**
- 低トラフィック時間帯での作業推奨
